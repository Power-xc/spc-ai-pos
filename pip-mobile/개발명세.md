## 코딩 규칙

### 컴포넌트 작성

- 모든 컴포넌트는 **함수형 컴포넌트**로 작성
- 컴포넌트 파일명은 **PascalCase** 사용 (예: `UserProfile.tsx`)
- 라우팅은 `react-router`를 사용
- 모든 컴포넌트는 클라이언트 사이드에서 동작 (서버 컴포넌트 없음)

### TypeScript

- `any` 타입 사용 **절대 금지**
- 모든 props는 `interface`로 타입 정의
- `types/` 폴더에 공통 타입 정의 관리

### 스타일링

- 인라인 스타일 사용 **금지** → Tailwind 클래스만 사용
- 반응형은 모바일 퍼스트 기준으로 작성 (`sm:` → `md:` → `lg:`)
- 색상, 폰트 등 디자인 토큰은 `tailwind.config.ts`에서 관리

### 임포트 순서

```typescript
// 1. React
import { useState } from "react";
import { Link } from "react-router";

// 2. 외부 라이브러리
import { Button } from "@/components/ui/button";

// 3. 내부 컴포넌트/훅/유틸
import { useUserStore } from "@/stores/userStore";
import { formatDate } from "@/lib/utils";

// 4. 타입
import type { User } from "@/types";
```

## 🔌 API 연동 규칙

> ⚠️ **모든 데이터는 반드시 API를 통해 가져와야 합니다. 컴포넌트에 데이터를 하드코딩하는 것은 절대 금지입니다.**

### 기본 원칙

- **컴포넌트 내 데이터 하드코딩 절대 금지** → 모든 데이터는 `src/lib/api.ts`를 통해 가져오기
- 실제 API가 없는 경우 **Mock API 방식으로 작업** → `src/lib/api.ts`에 가상 데이터 작성 후 연결
- Mock API 구조는 실제 API로 교체할 수 있도록 함수 시그니처를 동일하게 유지

### Mock API 작업 방식

```typescript
// src/lib/api.ts 에서 Mock 데이터 관리
function mockDelay<T>(data: T, ms = 300): Promise<T> {
  return new Promise((resolve) => setTimeout(() => resolve(data), ms));
}

// 실제 API 전환 시 이 함수 내부만 교체
export function getXxx(): Promise<XxxType[]> {
  return mockDelay(mockXxxData); // → fetch('/api/xxx').then(r => r.json())
}
```

### 컴포넌트 데이터 사용 방식

```tsx
// ✅ 올바른 방법: API에서 데이터 가져오기
const [data, setData] = useState<XxxType[]>([]);
useEffect(() => {
  getXxx().then(setData);
}, []);

// ❌ 금지: 컴포넌트 내 하드코딩
const data = [{ id: 1, name: "..." }];
```

### 추가 규칙

- **중복된 ID 값 사용 절대 금지** → API 요청/응답에서 동일한 ID가 중복으로 사용되지 않도록 항상 확인
- ID 생성 시 `uuid` 또는 서버에서 발급된 고유 ID만 사용 (임의의 숫자/문자 조합 금지)
- 리스트 렌더링 시 `key` prop에 **인덱스(index) 사용 금지** → 반드시 고유 ID 사용

  ```tsx
  // ❌ 금지
  items.map((item, index) => <Item key={index} />);

  // ✅ 올바른 방법
  items.map((item) => <Item key={item.id} />);
  ```

- API 엔드포인트는 `src/lib/api.ts`에서 **중앙 관리** (각 컴포넌트에서 직접 fetch 금지)
- 모든 API 호출은 **에러 처리** 필수 (try/catch 또는 에러 바운더리)
- API 응답 타입은 반드시 `src/types/` 폴더에 **인터페이스로 정의**
